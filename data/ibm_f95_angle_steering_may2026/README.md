# IBM Kingston F95 Angle-Steering Run, May 2026

Raw hardware data from the first deliberate-phase-injection run on IBM Kingston (Heron r2), testing whether the [F95](../../docs/ANALYTICAL_FORMULAS.md#f95) complex-C╬© angle is operationally **steerable** (not merely observable). The natural Kingston Z-detuning is small and uncontrolled; this run injects a deliberate per-chunk RZ phase at rate ╬® during the delay, expanding the existing [CPSI_COMPLEX_PLANE](../../experiments/CPSI_COMPLEX_PLANE.md) Februar/April 2026 framework into an active-steering experiment.

This run is registered as the typed Confirmation [`f95_angle_steering_kingston_may2026`](../../compute/RCPsiSquared.Core/Confirmations/ConfirmationsRegistry.cs).

## Experiment summary

- **Backend:** ibm_kingston (Heron r2, 156 qubits)
- **Date:** 2026-05-16 (both runs ~one minute apart in the same session)
- **Protocol:** Bell-like prep on each pair (H, S, CX), per-chunk RZ(╬®┬Ę╬öt) injection during the delay (the "steering" gate), 2-qubit state tomography with 9 Pauli bases at 2048 shots each
- **Two ╬® values:** 0.13 and 0.25 rad/╬╝s (the second is approximately 2├Ś the first; designed to verify linear ╬® scaling)
- **Total billed QPU:** ~14 minutes (both runs combined)
- **Job IDs:** `bxyj5yd4j` (╬®=0.13) and `bzklqwt7f` (╬®=0.25)
- **Drift between pre- and post-run calibration:** monitored per run (`cal_snapshot_start` / `cal_snapshot_end` in the JSONs)

**Key qualitative findings:**

1. **╬öŽå = ╬®┬Ęt reproducible (Q1 from CPSI_COMPLEX_PLANE):** crossings occur at the predicted angle modulo a bounded natural drift contribution.

3. **Active steering works (Q3):** the same Lindblad + RZ model fits both ╬® values on the same pair without retuning. The braŌåöket Hermitian off-diagonal of Žü on the prepared Bell-like state carries a well-defined complex phase, addressable on Heron r2 by per-chunk RZ injection on the steering qubit.

## Integrity

- **Drift during session:** monitored via `cal_snapshot_start` and `cal_snapshot_end` JSON blocks; small-to-zero on the run-day chain qubits (Heron r2 single-session calibration is stable).
- **Queue bias:** none (both runs at similar queue depth).
- **Readout:** not mitigated. IBM's default measurement error on Heron r2 is typically 1-3% per qubit.
- **Crossing detection precision:** linear interpolation between adjacent delays; the t_cross uncertainty is ~ ╬öt_delay / 2 Ōēł 1 ╬╝s at A_mid and ~2-3 ╬╝s at B_high (the latter with sparser delay grid).

The JSON files are the authoritative record. The PNGs are convenience renders from the run script.

## Reproducing the analysis

Zero QPU cost; the F97 lens reads from the JSON files in this directory:

```bash
python simulations/kingston_f97_lens.py
```

The script computes complex C╬©_com from the per-delay 4├Ś4 density matrices, tabulates magnitude / argument / F97 fixed-point magnitude along each trajectory, and finds the |c| = 1/4 crossings (matching the Confirmation entry's t_cross / arg_cross values exactly) plus the |c| = 1/2 crossings (none in this dataset; the Bell+-like initial state starts at the cusp magnitude). Three F97 framing findings drop out: (i) all trajectories live entirely inside the cardioid; (ii) the |c| = 1/4 crossings are off-cusp (arg Ōēł ŌłÆ85┬░ to ŌłÆ125┬░, not arg = 0); (iii) the F97 Half magnitude |c| = 1/2 is unreachable from this initial state.
