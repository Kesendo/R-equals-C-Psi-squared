# IBM Kingston Cusp-Slowing Run, April 2026

Raw hardware data from a Bell⁺ dwell-time experiment on IBM Kingston (Heron r2),
testing F57 (γ-invariance of K_dwell at the CΨ = 1/4 boundary).

## Experiment summary

- **Backend:** ibm_kingston (Heron r2, 156 qubits, 155 operational at run time)
- **Date:** 2026-04-16, 21:20 UTC
- **Protocol:** Bell⁺ state (H + CX) on two qubit pairs, native `qc.delay()` instruction for the idle period, 2-qubit state tomography with 9 Pauli bases at 2048 shots each
- **Queue at submit:** 0 pending jobs
- **Wall-clock:** ~5-6 minutes
- **Drift between pre- and post-run calibration:** 0.0% on all four qubits (same-session execution)

## Headline results

**γ-invariance (F57):** K_dwell / δ

- Pair A: 0.6492
- Pair B: 0.6937
- Ratio A/B: 0.936 (F57 predicts 1.000)
- Spread: 6.3% despite γ varying by factor 2.55

**Hardware crossing times:**

- Pair A: t_cross_measured = 11.21 μs (Aer pre-scan predicted 12.70 μs; hardware ~12% earlier)
- Pair B: t_cross_measured = 22.68 μs (Aer pre-scan predicted 25.73 μs; hardware ~12% earlier)

The earlier-than-predicted crossing is consistent with Kingston's real free-induction decay being faster than the Hahn-echo T2 reported by `backend.properties()` (the Aer pre-scan used the echo value).

**Absolute prefactor:** hardware K_dwell/δ ≈ 0.67 rather than the pure-Z-dephasing Bell⁺ value 1.0801. The difference is the Kingston T1 amplitude damping adding a second decay channel, steepening the slope at the crossing.

## Interpretation

The run is analyzed in two frames:

- **1D framing (real CΨ, F57 γ-invariance):** covered in [CRITICAL_SLOWING_AT_THE_CUSP.md](../../experiments/CRITICAL_SLOWING_AT_THE_CUSP.md) and the hardware-extension notes therein.

Uses the saved density matrices to show that both pairs follow logarithmic spirals in the c-plane with opposite rotation directions (Kingston residual Z-detunings: Pair A ≈ −2.6°/μs, Pair B ≈ +1.57°/μs).
